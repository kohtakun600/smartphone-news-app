# AIニュースアプリ 要件定義書

## 1. プロジェクト概要
毎日自動で最新のAI/機械学習関連ニュースを収集・要約し、閲覧できる個人用Webアプリケーション（PWA）を構築する。
サーバーレスかつ低コスト（実質無料）での運用を目指し、GitHub ActionsとGitHub Pagesを活用する。

## 2. 目的
- 個人の情報収集効率化。
- AIトレンドの早期把握。
- 低コスト・低運用負荷でのシステム維持。

## 3. システム構成技術
| カテゴリ | 技術スタック | 備考 |
| --- | --- | --- |
| **Frontend** | HTML5, CSS3, JavaScript (Vanilla) | GitHub Pagesでホスティング。PWA対応。 |
| **Backend** | Node.js | データ収集・加工スクリプト。 |
| **Infrastructure** | GitHub Actions | 定時実行（CRON）。 |
| **Database** | JSON Files | GitHubリポジトリ内にデータを保存。 |
| **News Source** | NewsAPI | 無料枠を利用。 |
| **AI Engine** | Google Gemini 1.5 Flash | 高速・低コスト要約。 |

## 4. 機能要件

### 4.1 ニュース収集・要約機能
- **実行タイミング**: 毎日 07:30 (JST) に自動実行。
- **収集対象**: NewsAPIを使用し、「Artificial Intelligence」「Machine Learning」などのキーワードで最新記事を収集。
- **フィルタリング**: 上位20件を選定（APIのスコアや独自ロジックに基づく）。
- **要約生成**: Gemini APIを使用し、各記事を日本語で3行程度に要約する。
- **データ保存**: 収集・要約結果を `data/latest_news.json` および `archives/yyyy-mm-dd.json` として保存。

### 4.2 ニュース閲覧機能 (Frontend)
- **記事一覧表示**: 最新の20件をカード形式で表示。
  - タイトル
  - 日本語要約
  - 元記事へのリンク
  - 発行日時
- **過去ログ閲覧**: 過去のニュースアーカイブへのアクセス（オプション）。

### 4.3 PWA機能
- **ホーム画面追加**: スマートフォンのホーム画面にアイコンを追加し、ネイティブアプリのように起動可能にする。
- **オフライン対応**: Service Workerを使用し、直近のデータはオフラインでも閲覧可能にする（簡易実装）。

## 5. 非機能要件
- **運用コスト**: 原則無料（Free Tier活用）。
- **保守性**: 複雑なフレームワークを使用せず、素のJS/HTML/CSSで構成し、ブラックボックス化を防ぐ。
- **信頼性**: GitHub Actionsの通知機能を使い、実行失敗時にメール通知を受け取る。

## 6. 制限事項
- 外部への公開やマルチユーザー対応は行わない。
- 認証機能は設けない（URLを知っている人のみが閲覧可能、またはプライベートリポジトリ運用の場合はPagesの閲覧権限に準拠）。
